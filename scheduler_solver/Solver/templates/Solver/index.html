How does it work?

Users

Full experience can be experienced only when user is logged in. But, core functionalities do not require user to be sign in. Ater sign in user's profile carries informations and data about his trials in different semesters. Unregistered user can't create content, but can visit all created data and generate corresponding time schedules.

Data

For each semester(Term) are uploaded items_json, which are working as templates. Users can steal these templates as json, or through internal functions of project. Registered users can changing preferencies about their selection. In case of not satysfying sample, users can upload custom set via json file.

Preferencies can be changed via form or via downloading and ten uploading json file.
